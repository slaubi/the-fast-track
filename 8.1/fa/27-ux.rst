واداشتن وب‌سایت به رفتاری شبیه یک SPA
===========================================================

.. index::
    single: Symfony UX
    single: Turbo
    single: Stimulus

وب‌سایت سریع است، اما هر کلیک همچنان یک صفحه‌ی HTML کامل را بازبارگیری می‌کند. پاسخ سنتی، ساخت یک *اپلیکیشن تک‌صفحه‌ای (SPA)* است: یک اپلیکیشن JavaScript که در پشته‌ای دیگر نوشته می‌شود و با API صحبت می‌کند. این یعنی یک اپلیکیشن دوم که باید توسعه، امن، مستقر و با اپلیکیشن اصلی هماهنگ نگه داشته شود.

سیمفونی پاسخ متفاوتی دارد. *Symfony UX* یک ابتکار است که تجربه‌ی SPA را به اپلیکیشن‌های سرورْرِندرشده می‌آورد: همچنان قالب‌های Twig و کنترلرهای سیمفونی بنویسید و تنها جایی که ارزش می‌افزاید، اندکی JavaScript بپاشید.

کشف Symfony UX
------------------------------

خبر خوب؟ ما از همان نخستین گام این کتاب از Symfony UX استفاده کرده‌ایم. شالوده‌ی webapp با دو بسته‌ی آن می‌آید: ``symfony/stimulus-bundle`` و ``symfony/ux-turbo``. به ``importmap.php`` نگاهی بیندازید:

.. code-block:: php
    :class: ignore

    return [
        'app' => ['path' => './assets/app.js', 'entrypoint' => true],
        '@hotwired/stimulus' => ['version' => '3.2.2'],
        '@symfony/stimulus-bundle' => ['path' => './vendor/symfony/stimulus-bundle/assets/dist/loader.js'],
        '@hotwired/turbo' => ['version' => '8.0.23'],
    ];

`Stimulus`_ یک چارچوب کوچک JavaScript است که *کنترلرهای* JavaScript را از طریق attributeهای ``data-`` به عناصر HTML متصل می‌کند. فایل ``assets/stimulus_bootstrap.js`` اپلیکیشن Stimulus را راه‌اندازی کرده و به صورت خودکار هر کنترلری را که در پوشه‌ی ``assets/controllers/`` ذخیره شده باشد، ثبت می‌کند.

`Turbo`_ بر روی این شالوده بنا می‌شود تا پیمایش را آنی کند.

پیمایش آنی با Turbo
--------------------------------------

.. index::
    single: Turbo;Drive

Turbo *Drive* در تمام این مدت بی‌صدا در حال کار بوده است: هر کلیک روی پیوند و ارسال فرم را رهگیری می‌کند، صفحه را در پس‌زمینه واکشی کرده و ``<body>`` را بدون بازبارگیری کامل صفحه جایگزین می‌کند. مرورگر همان JavaScript و CSS را زنده نگه می‌دارد، بنابراین پیمایش آنی به نظر می‌رسد.

ابزارهای توسعه‌دهنده‌ی مرورگر را باز کنید، به تب «Network» بروید و در وب‌سایت کلیک کنید: تغییرات صفحه، درخواست‌های ``fetch`` هستند، نه بارگیری کامل سند.

Turbo در ازای آن تنها یک چیز می‌خواهد: معناشناسی HTTP درست برای فرم‌ها. یک ارسال موفق باید بازهدایت کند (ارسال ما به لطف ``redirectToRoute()`` این کار را می‌کند) و یک ارسال ناموفق باید از یک کد وضعیت ``4xx`` استفاده کند. سیمفونی مورد دوم را به صورت خودکار رسیدگی می‌کند: زمانی که یک فرم ارسال‌شده نامعتبر است، ``render()`` با کد وضعیت ``422 Unprocessable Entity`` پاسخ می‌دهد.

به‌روزرسانی بخشی از یک صفحه با Turbo Frames
------------------------------------------------------------------------------

.. index::
    single: Turbo;Frame

Turbo Drive از بازبارگیری کامل صفحه جلوگیری می‌کند. Turbo *Frames* یک گام فراتر می‌رود: به یک قطعه از صفحه اجازه می‌دهد تا مستقل از باقی آن پیمایش کند.

فهرست کامنت‌ها یک نامزد عالی است. هنگام مرور صفحات کامنت از طریق پیوندهای «Previous» و «Next»، تنها فهرست باید تغییر کند؛ رِرِندرکردن سربرگ، فرم و پاورقی کاری بیهوده است. کامنت‌ها را در یک frame در بر بگیرید:

.. code-block:: diff
    :caption: patch_file

    --- i/templates/conference/show.html.twig
    +++ w/templates/conference/show.html.twig
    @@ -17,3 +17,4 @@
         <div class="row">
             <div class="col-12 col-lg-8">
    +            <turbo-frame id="comments" data-turbo-action="advance">
                 {% if comments|length > 0 %}
    @@ -56,5 +57,6 @@
                     No comments have been posted yet for this conference.
                 </div>
             {% endif %}
    +            </turbo-frame>
             </div>
             <div class="col-12 col-lg-4">

پیوندهای درون یک عنصر ``<turbo-frame>`` تنها همان frame را به‌روزرسانی می‌کنند: هنگام کلیک روی «Next»، Turbo صفحه را در پس‌زمینه واکشی کرده و frame منطبق را جایگزین می‌کند و باقی صفحه را دست‌نخورده می‌گذارد.

attribute‌ی ``data-turbo-action="advance"`` پیمایش frame را به یک بازدید کامل ارتقا می‌دهد: URL موجود در نوار آدرس به‌روزرسانی می‌شود (شامل پارامتر پرس‌وجوی ``offset``)، بنابراین کامنت‌های صفحه‌بندی‌شده قابل اشتراک‌گذاری می‌مانند و دکمه‌ی بازگشت مرورگر مطابق انتظار کار می‌کند.

پیش‌نمایش عکس کامنت با Stimulus
-------------------------------------------------------

.. index::
    single: Stimulus;Controller

زمان نوشتن نخستین خطوط JavaScript در کل کتاب فرا رسیده است. شرکت‌کنندگانی که عکسی ارسال می‌کنند، نمی‌توانند آن را پیش از ارسال کامنت ببینند. بیایید هنگامی که یک فایل انتخاب می‌کنند، یک پیش‌نمایش نمایش دهیم.

یک کنترلر Stimulus ایجاد کنید (نام فایل، نام کنترلر را تعیین می‌کند، یعنی ``photo-preview``):

.. code-block:: javascript
    :caption: assets/controllers/photo_preview_controller.js

    import { Controller } from '@hotwired/stimulus';

    export default class extends Controller {
        show() {
            const [photo] = this.element.files;
            if (!photo) {
                this.preview?.remove();
                this.preview = null;

                return;
            }

            if (!this.preview) {
                this.preview = document.createElement('img');
                this.preview.className = 'img-thumbnail mt-2';
                this.preview.style.maxHeight = '150px';
                this.element.insertAdjacentElement('afterend', this.preview);
            }

            this.preview.src = URL.createObjectURL(photo);
        }
    }

این تمام JavaScriptی است که نیاز داریم. Stimulus کنترلر را به صورت خودکار کشف و ثبت می‌کند. آن را از طریق attributeهای ``data-`` به فیلد عکس فرم کامنت متصل کنید:

.. code-block:: diff
    :caption: patch_file

    --- i/src/Form/CommentType.php
    +++ w/src/Form/CommentType.php
    @@ -26,6 +26,10 @@ class CommentType extends AbstractType
                 ->add('photo', FileType::class, [
                     'required' => false,
                     'mapped' => false,
    +                'attr' => [
    +                    'data-controller' => 'photo-preview',
    +                    'data-action' => 'change->photo-preview#show',
    +                ],
                     'constraints' => [
                         new Image(maxSize: '1024k')
                     ],

attribute‌ی ``data-controller`` کنترلر را به ورودی فایل متصل می‌کند و ``data-action`` هرگاه مقدار ورودی تغییر کند، متد ``show()`` آن را فراخوانی می‌کند. روی یک صفحه‌ی کنفرانس عکسی انتخاب کنید و از پیش‌نمایش لذت ببرید.

کنترلرهای Stimulus دست در دست Turbo کار می‌کنند: از آنجایی که Drive و Frames صفحه را بدون بازبارگیری به‌روزرسانی می‌کنند، کنترلرها به صورت خودکار با ورود و خروج عناصر از DOM متصل و قطع می‌شوند.

اپلیکیشن‌های موبایل چطور؟
---------------------------------------------------

ما تجربه‌ی SPA را بدون ساخت یک اپلیکیشن دوم به دست آوردیم: بدون پشته‌ی JavaScript جداگانه، بدون وب سرور دوم، بدون پیکربندی CORS، بدون هیچ چیز جدیدی برای استقرار.

و اگر روزی به یک اپلیکیشن موبایل *بومی (native)* نیاز داشتید، APIای که در گام قبلی ایجاد شد، مدخل درستی است: عمومی، مستندشده و مستقل از چارچوب است. وب‌سایت و اپلیکیشن موبایل می‌توانند به صورت مستقل تکامل یابند در حالی که بک‌اند یکسانی را به اشتراک می‌گذارند.
